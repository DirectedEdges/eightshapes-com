Compress

1

As the viewport width decreases, fluid columns compress to have smaller widths in absolute terms. This fluidity is the presumed default behavior for all grid columns. The rest of the behaviors are exceptions to this default.

Resist

2

Many of the components we work with have a minimum viable width around 200px. In a multi-column layout, these components will resist compressing below that minimum. Adjust the grid proportions proactively at smaller widths to accommodate these components.

Resist

3

Watch this column resist compression between 600px and 959px. Compare to columns 5 and 6 below, which do not resist.

Stack

4

As the viewport width decreases, breakpoints stack grid columns into a single column of components. This can happen all at once, or may happen at different breakpoints for different parts of the grid. See push and pull for ways to change the display order.

Push

7

In the HTML source order, this component comes before #8. It pushes after #8 at larger screen widths.

Pull

8

Push and pull are methods for creating a logical visual order for columns in a row that is different from the underlying HTML source order. For example, the natural position for the last component is on the right, but sometimes the designer wants to pull it to the left. Push and pull usually only work on multi-column layouts.

  • Pull forces a column to appear earlier visually, usually on the left of the screen.
  • Push forces a column to appear later, usually on the right.

Assemble

9

At a certain viewport width, assemble components from existing columns into a new column. The column is inert at other viewport widths, having no effect on the layout.

Assemble

10

Between 600px and 786px this component and #11 are assembled into a new column.

Disappear

12

At smaller sizes, some components become unnecessary or redundant. If the component was the only item in a column, that column should disappear. Usually, a disappearing column allows the column next to it to explode to fill the gap.

Explode

13

When a column disappears, the column next to it may explode to fill the remaining viewport width. For components that need more room, explode them to full width and stack the other column(s) in the row.

Below 960px, this column causes #12 to disappear, exploding to fill its space.

Lock

14

Some content has a maximum viable width, particularly if it contains paragraphs of text. Lock its maximum width to keep line lengths appropriate to the text size.

This column locks at 600px and stays centered.

Bleed

15

The visual appearance of certain components requires them to bleed over grid margins, usually at the smallest sizes. Bleed components with backgrounds or horizontal rules to extend those elements to the viewport edge.

This column bleeds to the viewport edges from 0px to 599px viewport width.

Parfait

16

A parfait row bleeds visually to the viewport edges while constraining the columns/components it contains to a page bed. Think of the parfait row as simple outer grid, with another grid inside it.

Parfait

17

A parfait row may have a different page bed width than the rest of the grid.

Apron

19

A parfait row may have a maximum width, creating an apron to frame the inner component grid. This controls the entire row, including the page bed. This apron example is orange.

This component is inside the following:

  • Outer parfait (dark gray, 100% width)
  • Inner apron (orange, 68em max-width)
  • Page bed + inner grid (light components, 62em max-width)

Help Using this Demo

Using a mobile device? Try viewing this page through an emulator if you want to see more than one size.

To see how the grid is put together, use your browser's inspector. I used SUSY grid as our base for the demo, but there are a number of grid frameworks capable of similar results. The CSS for this demo is compiled, so you will get a better view through inspector than looking at the CSS files directly.

It's worth mentioning that this demo grid is not efficiently built. In the interest of clarity, there are more layers and declarations than strictly necessary.
